# Working Today Work Dates Implementation Plan

## Purpose

Implement the Tasks Screen "Working Today" button with a normalized work-date table.

When a user clicks "Working Today", the backend inserts one row into `WORK_ITEM_WORK_DATES` for the task and today's UTC date. When the user reverts the action, the backend removes that row. `WORK_ITEM_EVENTS` remains the audit trail for add/remove history.

This plan follows:

- `docs/backend-todo-list.md`, especially the `WORK_ITEM_WORK_DATES` schema, worked-date APIs, and test checklist.
- `skills/backend-senior-engineer/SKILL.md`, especially ownership checks, bind variables, explicit transactions, optimistic locking, audit events, stable API responses, and test coverage.

## Source Of Truth

Use `WORK_ITEM_WORK_DATES` as the active source of truth for whether a task is marked as worked/planned on a date.

Do not store comma-separated dates in `WORK_ITEMS`. The frontend can still receive `worked_dates` as an array, but that array is derived from rows in `WORK_ITEM_WORK_DATES`.

## Functional Requirements

1. The Tasks Screen calls the backend when the user clicks "Working Today".
2. If the request marks the task as working today, insert today's UTC date into `WORK_ITEM_WORK_DATES`.
3. If the work-date row already exists, treat the request as idempotent success.
4. If the request reverts the action, delete today's UTC date row from `WORK_ITEM_WORK_DATES`.
5. If the row being removed is absent, treat the request as idempotent success.
6. Return `worked_dates` as an array derived from `WORK_ITEM_WORK_DATES`.
7. Return `working_today` as a derived boolean based on row existence for today's UTC date.
8. Insert a `WORK_ITEM_EVENTS` audit row for add/remove actions that change the active state.

## Database Assumptions

`WORK_ITEMS` exists with at least:

| Column | Requirement |
| --- | --- |
| `TASK_ID` | Primary key. |
| `USER_ID` | Owner FK; used for row ownership. |
| `UPDATED_AT` | Set when the active work-date state changes. |
| `ROW_VERSION` | Incremented on active work-date state changes for optimistic locking. |

`WORK_ITEM_WORK_DATES` exists with at least:

| Column | Requirement |
| --- | --- |
| `WORK_ITEM_WORK_DATE_ID` | PK generated by `WORK_ITEM_WORK_DATES_SEQ.NEXTVAL`. |
| `USER_ID` | FK to `APP_USERS`; owner and query partition. |
| `TASK_ID` | FK to `WORK_ITEMS`. |
| `WORK_DATE` | `DATE`; today's UTC date for this scope. |
| `SOURCE` | `USER`, `AI_QUEST`, `IMPORT`, or `SYSTEM`. |
| `PLANNED_MINUTES` | Optional. |
| `ACTUAL_MINUTES` | Optional. |
| `NOTES` | Optional per-day work notes. |
| `CREATED_AT` | Insert timestamp. |
| `UPDATED_AT` | Update timestamp. |
| `ROW_VERSION` | Row version for future per-day edits. |

Required constraints and indexes:

```sql
CONSTRAINT WORK_ITEM_WORK_DATES_UK UNIQUE (USER_ID, TASK_ID, WORK_DATE)
```

```sql
CREATE INDEX WORK_ITEM_WORK_DATES_USER_DATE_IDX
  ON WORK_ITEM_WORK_DATES(USER_ID, WORK_DATE);

CREATE INDEX WORK_ITEM_WORK_DATES_TASK_DATE_IDX
  ON WORK_ITEM_WORK_DATES(TASK_ID, WORK_DATE);
```

`WORK_ITEM_EVENTS` exists with at least:

| Column | Requirement |
| --- | --- |
| `EVENT_ID` | Generated by `WORK_ITEM_EVENTS_SEQ.NEXTVAL`. |
| `TASK_ID` | FK to `WORK_ITEMS`. |
| `USER_ID` | Owner FK. |
| `EVENT_TYPE` | Use `WORKING_TODAY_UPDATED`. |
| `OLD_VALUE_JSON` | JSON diff/snapshot before the change. |
| `NEW_VALUE_JSON` | JSON diff/snapshot after the change. |
| `CREATED_AT` | Insert timestamp. |

Oracle DB, `oracledb`, and OCI SDK/model dependencies are assumed to be available. This feature does not need an OCI GenAI call.

## API Contract

### Endpoint

`PUT /api/v1/tasks/{task_id}/today`

### Request

```json
{
  "is_working_today": true,
  "row_version": 4
}
```

Fields:

| Field | Type | Required | Notes |
| --- | --- | --- | --- |
| `is_working_today` | boolean | yes | `true` inserts today's UTC date row; `false` deletes it. |
| `row_version` | integer | yes | Required for durable task state changes. |

Do not accept `user_id` from the body. Resolve the current user from auth context or the temporary user resolver.

For automated tests only, the service layer may accept an injected clock so today's UTC date can be fixed without exposing a public `work_date` override.

### Response

```json
{
  "data": {
    "task_id": 1002,
    "work_date": "2026-05-05",
    "working_today": true,
    "worked_dates": ["2026-05-01", "2026-05-05"],
    "row_version": 5
  },
  "meta": {
    "request_id": "2f7bd7e7-8928-41c0-88c4-d8f1d4df0f6f"
  }
}
```

## Date Rules

Use today's date in UTC for this scope, not browser local time.

```python
from datetime import UTC, datetime

def utc_today_iso() -> str:
    return datetime.now(UTC).date().isoformat()
```

If the product later switches to user-local productivity dates, update this API contract, service utility, tests, and overview queries together.

### Repository Stubs

```python
# backend/app/repositories/work_items.py
import json
from typing import Any


def fetch_work_item_for_update(conn, *, task_id: int, user_id: int) -> dict[str, Any] | None:
    cursor = conn.cursor()
    cursor.execute(
        """
        SELECT TASK_ID, USER_ID, ROW_VERSION
        FROM WORK_ITEMS
        WHERE TASK_ID = :task_id
          AND USER_ID = :user_id
        FOR UPDATE
        """,
        {"task_id": task_id, "user_id": user_id},
    )
    row = cursor.fetchone()
    if row is None:
        return None
    return {"task_id": row[0], "user_id": row[1], "row_version": row[2]}


def work_date_exists(conn, *, task_id: int, user_id: int, work_date: str) -> bool:
    cursor = conn.cursor()
    cursor.execute(
        """
        SELECT 1
        FROM WORK_ITEM_WORK_DATES
        WHERE TASK_ID = :task_id
          AND USER_ID = :user_id
          AND WORK_DATE = TO_DATE(:work_date, 'YYYY-MM-DD')
        """,
        {"task_id": task_id, "user_id": user_id, "work_date": work_date},
    )
    return cursor.fetchone() is not None


def insert_work_date_if_absent(conn, *, task_id: int, user_id: int, work_date: str) -> bool:
    cursor = conn.cursor()
    cursor.execute(
        """
        INSERT INTO WORK_ITEM_WORK_DATES (
            WORK_ITEM_WORK_DATE_ID,
            USER_ID,
            TASK_ID,
            WORK_DATE,
            SOURCE,
            CREATED_AT,
            UPDATED_AT,
            ROW_VERSION
        )
        SELECT
            WORK_ITEM_WORK_DATES_SEQ.NEXTVAL,
            :user_id,
            :task_id,
            TO_DATE(:work_date, 'YYYY-MM-DD'),
            'USER',
            SYSTIMESTAMP,
            SYSTIMESTAMP,
            1
        FROM DUAL
        WHERE NOT EXISTS (
            SELECT 1
            FROM WORK_ITEM_WORK_DATES
            WHERE USER_ID = :user_id
              AND TASK_ID = :task_id
              AND WORK_DATE = TO_DATE(:work_date, 'YYYY-MM-DD')
        )
        """,
        {"task_id": task_id, "user_id": user_id, "work_date": work_date},
    )
    return cursor.rowcount == 1


def delete_work_date_if_present(conn, *, task_id: int, user_id: int, work_date: str) -> bool:
    cursor = conn.cursor()
    cursor.execute(
        """
        DELETE FROM WORK_ITEM_WORK_DATES
        WHERE TASK_ID = :task_id
          AND USER_ID = :user_id
          AND WORK_DATE = TO_DATE(:work_date, 'YYYY-MM-DD')
        """,
        {"task_id": task_id, "user_id": user_id, "work_date": work_date},
    )
    return cursor.rowcount == 1


def touch_work_item(conn, *, task_id: int, user_id: int, expected_row_version: int) -> int | None:
    cursor = conn.cursor()
    new_row_version = cursor.var(int)
    cursor.execute(
        """
        UPDATE WORK_ITEMS
        SET UPDATED_AT = SYSTIMESTAMP,
            ROW_VERSION = ROW_VERSION + 1
        WHERE TASK_ID = :task_id
          AND USER_ID = :user_id
          AND ROW_VERSION = :row_version
        RETURNING ROW_VERSION INTO :new_row_version
        """,
        {
            "task_id": task_id,
            "user_id": user_id,
            "row_version": expected_row_version,
            "new_row_version": new_row_version,
        },
    )
    if cursor.rowcount != 1:
        return None
    return new_row_version.getvalue()[0]


def list_worked_dates(conn, *, task_id: int, user_id: int) -> list[str]:
    cursor = conn.cursor()
    cursor.execute(
        """
        SELECT TO_CHAR(WORK_DATE, 'YYYY-MM-DD')
        FROM WORK_ITEM_WORK_DATES
        WHERE TASK_ID = :task_id
          AND USER_ID = :user_id
        ORDER BY WORK_DATE
        """,
        {"task_id": task_id, "user_id": user_id},
    )
    return [row[0] for row in cursor.fetchall()]


def insert_working_today_event(
    conn,
    *,
    task_id: int,
    user_id: int,
    work_date: str,
    is_working_today: bool,
    changed: bool,
) -> None:
    cursor = conn.cursor()
    cursor.execute(
        """
        INSERT INTO WORK_ITEM_EVENTS (
            EVENT_ID,
            TASK_ID,
            USER_ID,
            EVENT_TYPE,
            OLD_VALUE_JSON,
            NEW_VALUE_JSON,
            CREATED_AT
        )
        VALUES (
            WORK_ITEM_EVENTS_SEQ.NEXTVAL,
            :task_id,
            :user_id,
            'WORKING_TODAY_UPDATED',
            :old_value_json,
            :new_value_json,
            SYSTIMESTAMP
        )
        """,
        {
            "task_id": task_id,
            "user_id": user_id,
            "old_value_json": json.dumps({"work_date": work_date}, separators=(",", ":")),
            "new_value_json": json.dumps(
                {
                    "work_date": work_date,
                    "is_working_today": is_working_today,
                    "changed": changed,
                },
                separators=(",", ":"),
            ),
        },
    )
```

### Service Stub

```python
# backend/app/services/tasks.py
from app.repositories.work_items import (
    delete_work_date_if_present,
    fetch_work_item_for_update,
    insert_work_date_if_absent,
    insert_working_today_event,
    list_worked_dates,
    touch_work_item,
)
from app.services.work_dates import get_today_utc_date


class NotFoundError(Exception):
    pass


class RowVersionConflictError(Exception):
    pass


def set_working_today(conn, *, task_id: int, user_id: int, request):
    work_date = get_today_utc_date()

    try:
        task = fetch_work_item_for_update(conn, task_id=task_id, user_id=user_id)
        if task is None:
            raise NotFoundError("Task not found")

        if task["row_version"] != request.row_version:
            raise RowVersionConflictError("Task was updated by another request")

        if request.is_working_today:
            changed = insert_work_date_if_absent(
                conn,
                task_id=task_id,
                user_id=user_id,
                work_date=work_date,
            )
        else:
            changed = delete_work_date_if_present(
                conn,
                task_id=task_id,
                user_id=user_id,
                work_date=work_date,
            )

        row_version = task["row_version"]
        if changed:
            new_row_version = touch_work_item(
                conn,
                task_id=task_id,
                user_id=user_id,
                expected_row_version=task["row_version"],
            )
            if new_row_version is None:
                raise RowVersionConflictError("Task was updated by another request")
            row_version = new_row_version

            insert_working_today_event(
                conn,
                task_id=task_id,
                user_id=user_id,
                work_date=work_date,
                is_working_today=request.is_working_today,
                changed=changed,
            )

        worked_dates = list_worked_dates(conn, task_id=task_id, user_id=user_id)
        conn.commit()

        return {
            "task_id": task_id,
            "work_date": work_date,
            "working_today": work_date in worked_dates,
            "worked_dates": worked_dates,
            "row_version": row_version,
        }
    except Exception:
        conn.rollback()
        raise
```

## Read-Side Queries

Daily work:

```sql
SELECT w.*
FROM WORK_ITEM_WORK_DATES d
JOIN WORK_ITEMS w ON w.TASK_ID = d.TASK_ID
WHERE d.USER_ID = :user_id
  AND d.WORK_DATE = TO_DATE(:work_date, 'YYYY-MM-DD')
ORDER BY w.PRIORITY, w.DUE_DATE
```

Weekly overview inputs:

```sql
SELECT
  d.WORK_DATE,
  COUNT(DISTINCT d.TASK_ID) AS TASKS_WORKED,
  SUM(NVL(d.ACTUAL_MINUTES, 0)) AS WORK_DATE_MINUTES,
  SUM(NVL(w.XP_VALUE, 0)) AS XP_SCOPE
FROM WORK_ITEM_WORK_DATES d
JOIN WORK_ITEMS w ON w.TASK_ID = d.TASK_ID
WHERE d.USER_ID = :user_id
  AND d.WORK_DATE BETWEEN TO_DATE(:week_start, 'YYYY-MM-DD')
                      AND TO_DATE(:week_end, 'YYYY-MM-DD')
GROUP BY d.WORK_DATE
ORDER BY d.WORK_DATE
```

## Transaction Behavior

Perform the Working Today write in one transaction:

1. Resolve current user.
2. Resolve today's UTC date.
3. Lock the task row by `TASK_ID` and `USER_ID`.
4. Validate `ROW_VERSION`.
5. Insert or delete `WORK_ITEM_WORK_DATES`.
6. If the active state changed, update `WORK_ITEMS.UPDATED_AT` and increment `WORK_ITEMS.ROW_VERSION`.
7. If the active state changed, insert `WORK_ITEM_EVENTS`.
8. Read current `worked_dates` for the response.
9. Commit.

If any step fails, roll back the whole transaction. Do not persist the work-date insert/delete if the audit insert fails.

## Error Handling

| Case | Status | Error code |
| --- | --- | --- |
| Invalid request body | `422` | `VALIDATION_ERROR` |
| Task not found for current user | `404` | `TASK_NOT_FOUND` |
| Stale `row_version` | `409` | `ROW_VERSION_CONFLICT` |
| Oracle unique constraint race on insert | retry once or return current state | `WORK_DATE_ALREADY_EXISTS` only if not treated as idempotent |
| Oracle DB unavailable | `503` | `DATABASE_UNAVAILABLE` |

Do not return SQL text, stack traces, Oracle wallet paths, credentials, or raw provider errors to the frontend.

## Test Checklist

Unit tests:

- Mark Working Today inserts one `WORK_ITEM_WORK_DATES` row.
- Repeated mark is idempotent and does not duplicate the row.
- Revert deletes the matching `WORK_ITEM_WORK_DATES` row.
- Repeated revert is idempotent.
- `working_today` is derived from row existence.
- `worked_dates` is derived from table rows sorted by date.
- Stale `row_version` returns conflict.

Repository or integration tests:

- Insert uses `TASK_ID`, `USER_ID`, and `WORK_DATE`.
- Unique constraint prevents duplicate rows.
- Missing task for user returns not found.
- Audit insert occurs on successful add and remove.
- Audit insert failure rolls back the work-date change.
- Daily and weekly overview queries use `WORK_ITEM_WORK_DATES(USER_ID, WORK_DATE)`.
- Oracle bind variables are used for every SQL value.

API contract tests:

- `PUT /api/v1/tasks/{task_id}/today` with `is_working_today=true`.
- `PUT /api/v1/tasks/{task_id}/today` with `is_working_today=false`.
- Response includes `task_id`, `work_date`, `working_today`, `worked_dates`, and `row_version`.
- Request body cannot spoof `user_id`.

Frontend wiring tests after implementation:

- Tasks Screen button sends `is_working_today=true`.
- Revert action sends `is_working_today=false`.
- UI updates from the response instead of assuming success.
- Loading, success, conflict, and recoverable server-error states are visible.
